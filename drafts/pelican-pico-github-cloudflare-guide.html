Building a Website with Pelican, Pico CSS, GitHub Pages, and Cloudflare

When I set out to build this web site I wanted to use freely available tools. This article shows you how to build your own web site, step by step, using the same tools. But first, a brief overview of the tools you’ll be using:

  • Pelican
    A powerful static site generator written in Python. It lets you write content in Markdown or reStructuredText, and generates a fast, customizable static website.

  • Pico CSS
    A minimal, classless CSS framework that makes your site look modern and responsive with almost no effort.

  • GitHub Pages
    Free hosting for static sites directly from your GitHub repository.

  • Cloudflare
    Domain registration, DNS management, and security, making your custom domain setup easy and robust.

1. Setting Up Pelican

  • Install Pelican: Follow the official installation guide. bash pip install pelican markdown
  • Create a New Project: bash pelican-quickstart Answer the prompts to set up your site.
  • Write Content: Add Markdown or reStructuredText files to your content/ directory. See Writing Content.

2. Integrating Pico CSS

  • Download Pico CSS: Download pico.min.css and place it in your theme’s static/css/ folder.
  • Link Pico CSS in Your Theme: Edit your base template (e.g., sitefiles/themes/jjriv/templates/base.html): html <link rel="stylesheet" href="{{ SITEURL }}/static/css/pico.min.css"> Or use the CDN: html <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@picocss/pico@2/css/pico.min.css">
  • Customize as Needed: Pico works out-of-the-box, but you can tweak your HTML for best results. See Pico CSS Docs.

3. Running the Development Server

  • Preview Locally: bash pelican --listen Or use: bash pelican --autoreload Visit http://localhost:8000 to see your site.

4. Publishing Your Site

  • Generate Output: bash pelican The static files will be in the output/ directory.
  • Automate Publishing: Use scripts or Pelican’s publish instructions.

5. Hosting on GitHub Pages

  • Create a GitHub Repository: Push your Pelican project to GitHub.
  • Configure GitHub Pages:
  • Go to your repo’s settings.
  • Under “Pages,” select the branch and folder (e.g., /output) to publish.
  • See GitHub Pages Docs.
  • Automate Deployment: Use GitHub Actions or a publish script to push the output/ folder to the gh-pages branch.

6. Setting Up a Custom Domain with Cloudflare

  • Register Your Domain: Use any domain registrar.

  • Add Your Domain to Cloudflare:

  • Create a Cloudflare account.

  • Add your domain and update your registrar’s nameservers to Cloudflare’s.

  • Configure DNS for GitHub Pages:

  • Add a CNAME record pointing your domain to <username>.github.io.
  • For www, add a CNAME for www to your root domain.

  • See Cloudflare DNS Docs and GitHub Pages Custom Domain.

  • Enable HTTPS: Cloudflare provides free SSL. Make sure “Always Use HTTPS” is enabled.
